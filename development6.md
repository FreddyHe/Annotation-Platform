基于上面的发现，我把方案收紧成这样：
数据流（最简版）
边端推理回传时：

推理完成 → 分池（现有逻辑不动）
所有 HIGH 和 LOW_A → 通过 importPredictions 进主项目（LS project lsProjectId）作为 prediction

训练时：如果人工没改，自动用 prediction 训练；人工改了，用 annotation 训练


LOW_B → 攒够 N 张（默认 100）→ 新建"增量项目"{主项目名}__增量XXX__待审

同时写入 inference_data_points.ls_task_id 和 ls_sub_project_id



人工审核完成后：

LS webhook 通知 Spring Boot → 根据 ls_task_id 找到 inference_data_points → 写入审核后的 bbox + humanReviewed=true
训练时 FormatConverterService.buildFeedbackDataset() 就能拿到这些审核后的 LOW_B 数据

训练触发时：

检查所有"增量项目"的审核状态（调 LS API num_tasks_with_annotations）
前端显示"用了哪些、跳过了哪些"
审完的增量项目：数据已经通过 webhook 回写到 inference_data_points，buildFeedbackDataset 自动就能拿到
